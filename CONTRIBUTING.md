# 参与 Rainrain / Contributing

谢谢你愿意帮 Rainrain 变得更好。这是一个面向社科研究者的、隐私优先的本地研究工具，欢迎各种形式的参与。

## 可以怎么帮

- **报 Bug** —— 最有价值。请在 [Issues](https://github.com/chinnkirsten/rainrain/issues) 里开一条，尽量写清：
  - 你的系统（Windows / macOS）和 Rainrain 版本；
  - 你做了什么、期望看到什么、实际看到什么；
  - 能复现的步骤，越具体越好；有截图/报错更佳。
  - ⚠️ **千万不要**把真实的研究资料、受访者信息、录音等贴进 issue（见下方「隐私红线」）。
- **提功能建议** —— 开一条 issue，讲清你的研究场景和痛点。真实的研究工作流描述，比「加个 XX 功能」更有用。
- **翻译 / 改文档** —— 界面文案、使用说明、README 的措辞改进都欢迎。
- **写代码** —— 见下方开发说明。动手前，较大的改动请先开 issue 聊一下方向，避免白做。

## 隐私红线（务必遵守）

Rainrain 的立身之本是「研究资料绝不上云」。因此：

- **绝不**把任何真实研究数据、受访者实名、访谈录音/转录、知情同意书等提交到本仓库、issue 或 PR。
- 演示/测试请用虚构数据（仓库自带的 Sam Avery「社区花园」示例即为此设）。
- 发现任何会把用户数据外传的行为，请视作安全问题优先报告。

## 开发说明

技术栈：Next.js + React + TypeScript + Electron（桌面壳）。

```bash
# 需要 Node.js 20+
npm install
npm run dev        # 浏览器里开发：http://localhost:3000
npm run electron   # 以桌面 App 形式运行
```

- 本地默认用磁盘存储（`storage/` 目录）；不配置任何云 token。
- 提交前请跑 `npm run lint`。
- 保持改动聚焦、提交信息清晰；PR 描述里说明「解决什么问题、怎么验证的」。
- 代码风格向现有文件看齐（命名、注释密度、写法）。
