# Rainrain · 使用说明

私人本地研究图书馆。界面为英文，下文每个英文按钮后附中文。所有资料只在你本机，永不上云。

目录：1 安装与首次启动 · 2 界面与导航 · 3 搜索与筛选 · 4 阅读 · 5 上传 · 6 编辑与删除 · 7 摘录证据/质性编码 · 8 受访者数据库 · 9 文献 · 10 笔记 · 11 导出摘要 · 12 备份与恢复 · 13 改密码与隐私 · 14 本地 AI(可选) · 15 常见问题/排错

## 1. 安装与首次启动

### Windows

- 双击 Rainrain-Setup-0.1.0.exe 安装。
- 首次打开弹「已保护你的电脑 / Windows protected your PC」→ 点 更多信息（More info） → 仍要运行（Run anyway）。这是因为程序未做数字签名，安全的。

### Mac（仅 Apple 芯片 M 系列）

- 打开 Rainrain-0.1.0-arm64.dmg，把 Rainrain 拖进「应用程序」。
- 首次打开若提示「已损坏，无法打开」——这是未签名应用的隔离标记，不是真坏。开「终端」粘贴回车：

```bash
xattr -dr com.apple.quarantine /Applications/Rainrain.app
```

- 再双击 Rainrain 即可打开。

### 通用

- 输入默认密码 rainrain 进入。
- 点右上角齿轮 → Settings 设置 → 改成自己的密码（只存本机）。

## 2. 界面与导航

- 首页 Research journey 研究历程：按研究阶段分卡片，点进去就是该阶段的馆藏;右侧「Edit structure 编辑结构」可加阶段/子课题。
- 顶部导航：All Collections 全部馆藏、Respondents 受访者、Readings 文献、Evidence 证据库、Notes 笔记、Summary 摘要、Help 帮助。
- 两个齿轮要分清：最右上角(导航栏)的齿轮 = 设置/改密码;馆藏页工具栏里的齿轮 = 备份 + 回收站（见第 12 节）。

## 3. 搜索 · 筛选 · 排序

- 搜索框：同时搜 标题、标签、以及文档正文全文（含扫描件 OCR 出来的字）。
- 简繁日通搜：输入简体也能命中繁体/日文写法，按相关度排序。
- 筛选：按类型（PDF/图片/录音/表格…）、年份、主题标签、收藏 ★ 过滤;右侧可切换网格/列表、改排序（最近/年份/标题/大小）。
- 快捷键：按 / 直接聚焦搜索框。

## 4. 阅读材料

- 点任意卡片打开查看器。PDF：底部有缩放（50–400%）和翻页（‹ Page X / N ›）;放大后可滚动看清。
- Word / 文本 / 访谈转录：内联显示，可直接用鼠标划选文字（用于摘录证据）。
- 图片可放大、录音/视频就地播放;两侧 ‹ › 切换上一项 / 下一项。

## 5. 上传材料

- 在任意馆藏页点 Upload 上传。
- 选「归入哪个阶段/子课题」，把文件拖进去（可一次多选，可加标签）。
- 上传后自动：生成封面、建全文索引（立刻可搜）;扫描件自动 OCR（仅 macOS;Windows 上文字版 PDF/Word/图片/录音都正常，只是新扫描件不自动 OCR）。

> 支持的文件类型：PDF、Word/RTF/txt/md、PPT、图片(jpg/png/tiff/heic…)、表格(xlsx/csv)、数据(geojson/json/kml…)、录音(mp3/m4a/wav…)、视频(mp4/mov…)。

## 6. 编辑与删除条目

- 打开某条目 → 点 Edit 编辑，可改标题、描述/笔记、年份、阶段、标签，以及著录信息（作者/出版/馆藏/索书号/版本，用于规范引用）。
- 删除 → 进回收站（不会立刻消失，可还原;见第 12 节）。示例条目也能删。

## 7. 摘录证据 / 质性编码（coding）

这是做访谈/文本分析的核心：把材料里的关键句摘出来、打上主题码，自动汇成编码本。

- 打开一份文档 → 用鼠标划选一段文字 → 浮出 Save as evidence 摘录为证据。
- 填主题码（标签，逗号分隔）、可加批注和页码 → 保存。证据自动带来源出处。
- PDF / 图片不能划选 → 用 “+ Add evidence” 手动输入引文。
- 顶部 Evidence 证据库 汇总所有摘录;切到 By theme 按主题 看编码本：每个码多少条、跨多少来源/受访者。
- 点 Export 导出 把引文导成带引用格式（BibTeX / RIS / CSV），可进 Zotero 或论文。

> 方法对应： 这套「划选→打码→按码检索→写作」直接吃 主题分析(thematic analysis)、扎根理论开放编码、框架分析。边界：轻量编码，没有多层码树、码间关系、双编码者信度——要这些再上 NVivo/Atlas.ti。

## 8. 受访者数据库（多项目 + 可自定义变量）

把访谈对象整理成可查询、可交叉分析的数据库;每个研究/田野是一个独立项目，自己定义要记录的变量。

### 8.1 选/建项目

- 顶部项目下拉 切换不同田野;点 + New project 新建项目 开一个新研究（先取个名）。

### 8.2 编辑变量（即「可编辑的变量」）

- 点 Edit variables 编辑变量。
- 每个变量可设：显示名、类型（类别 / 数值 / 文本）、是否可筛选。类别型 → 会变成分面筛选 + 交叉表维度;数值型如年龄;文本型如备注。
- 可增 / 删 / 改名 / 调顺序 → 保存。改完立刻反映到筛选和交叉表。

### 8.3 加/改受访者

- 点 + Add respondent 添加受访者，按当前项目的变量自动生成表单填写;点表格里某行可编辑或删除。

### 8.4 查询与分析

- 上方分面筛选（点标签即筛）;搜索框搜姓名/编号/任意字段。
- 交叉表：选 Rows × Columns 两个类别变量，实时出计数热力表（如 职业 × 地区）。
- 点某位受访者：完整档案 + 录音播放 + 转录稿 + Key quotes 关键引文（按访谈编号自动关联其名下的证据摘录）。

## 9. 文献（Readings）

- 顶部 Readings 文献：二手文献书目。标「已读/待读」、点 + Note 加批注、按子课题归类、或 + Add reading 新增。与一手史料分开管理。

## 10. 笔记（Notes · 扎根材料的双链）

- 顶部 Notes 笔记 → New note 新建。
- 正文输入 [[ 触发联想补全——可链到其它笔记 / 史料条目 / 受访者;↑↓ 选、回车插入。
- 链到一个还不存在的标题 = 留一条「待写」（左下To write 待写清单可见，点一下即新建那条笔记）。
- 点 Preview 预览 看渲染后的链接（可点击跳转）;每条笔记底部有出链 + 反向链接（谁链到了我）。

## 11. 导出摘要（Summary）

- 顶部 Summary 摘要：勾选要包含的章节（研究陈述/教育/论文…）→ 打印为 PDF 或 复制为文本。一页式简介，方便汇报/申请/给导师。

## 12. 备份与恢复

### 备份

- 到「全部馆藏」页 → 点工具栏的齿轮 ⚙ → Back up now 立即备份。
- 会在数据目录的 storage/backups/ 下生成一个带时间戳的 zip（全程本地）。

### 恢复 / 换电脑

- 完全退出应用。
- 文件管理器地址栏进数据目录：Windows %APPDATA%\Rainrain、Mac ~/Library/Application Support/Rainrain。
- 把旧 storage 改名留底，再把备份 zip 里的 storage 解压回来（覆盖）→ 重开应用即恢复。

> 回收站在哪： 也在「全部馆藏」工具栏的齿轮 ⚙ 里 → Recycle bin 回收站。删除的条目先进这里，可还原或彻底删除，不会立刻消失。

## 13. 改密码与隐私

- 改密码：最右上角（导航栏）齿轮 → Settings 设置 → 输当前密码 + 新密码。只存本机。
- 隐私：所有研究资料只在本机的数据目录，永不上传任何云;可选的 AI 摘要也在本机运行。

## 14. 本地 AI 摘要（可选）

- 装 Ollama（免费本地 AI 运行器）。
- 终端跑一次 ollama pull qwen2.5 拉一个模型（约几 GB）。
- 回到应用打开任意文档 → 侧栏出现 Summarize with AI 生成 AI 摘要。点一下，AI 在你电脑上运行、资料不出本机。没装则不显示/会提示如何启用。

## 15. 常见问题 / 排错（FAQ）

Mac 提示「已损坏，无法打开」 → 未签名应用的隔离标记。终端跑：

```bash
xattr -dr com.apple.quarantine /Applications/Rainrain.app　然后重开。或：先把应用拖进「应用程序」，再右键→打开。
```

Windows 蓝屏「Windows 已保护你的电脑」 → 点「更多信息」→「仍要运行」。未签名程序的正常提示。

忘了密码 → 进数据目录（见第 12 节）删掉 storage/auth.json，密码会还原成默认 rainrain，进去后重设。

某个文件打不开/没封面 → 多为不常见格式;仍可在条目里「下载」用本机软件打开。

受访者页空白 / 显示不对 → 若是从旧版升级，需用迁移后的 respondents.json + respondent-projects.json（见随附数据更新说明）。

「Summarize with AI」没出现 → 没装/没运行 Ollama，或没 pull 模型;见第 14 节。它本就是可选项。

换了新电脑 / 想搬数据 → 用第 12 节的备份+恢复，把 storage 整个搬过去即可。

先在线试用界面：https://rainrain-ten.vercel.app（密码 rainrain2026，只读演示）。应用内「Help / 帮助」页也有简版导览。
